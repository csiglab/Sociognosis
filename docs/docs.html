01 — Overview

What is Sociognosis?

Sociognosis aims to identify and formalize the set of epistemic elements that enable an agent to effectively navigate—through understanding and action—social reality.

At its core, the project stabilizes a generative system that produces a Domain Epistemic Artifact Set (DESA) representing social reality. This artifact set renders social phenomena intelligible as a structured domain of analysis and action, connecting abstract epistemic categories to real-world decision-making processes.

Cognoscere ad agendum. — To know in order to act. The project bridges the gap between understanding social structures and intervening within them.

02 — Core Concepts

Foundational Framework

Ontology & Epistemology

Ontology and epistemology are the right and left hand of knowledge construction—mutually constraining duals of the same epistemic act. Ontology defines what is; epistemology defines how it is accessed and represented.

DESA

The Domain Epistemic Artifact Set is the complete collection of instantiated epistemic objects—models, datasets, equations, graphs—produced to describe, explain, predict, and manipulate a specific domain of reality.

Reality Section

A Reality Section is a bounded segment of reality targeted by epistemic practice. It encompasses observable and unobservable phenomena, latent mechanisms, causal architectures, and state trajectories across time.

Generative System

The underlying system that produces and organizes the DESA. It is the subject of continuous study, refinement, and improvement—with emphasis on consistency, expressiveness, and explanatory power.

04 — Sub-Projects

Analytical Spaces

Sociognosis decomposes social reality into distinct analytical spaces, each rendering a specific dimension of the social world intelligible and comparable.

Index

Interactive

The central epistemic index—a force-directed graph visualization mapping the architecture of epistemic capabilities, categories, and their relationships.

  • Space Explorer (graph canvas)
  • Node Editor
  • JSON-backed data model

Product Space

Analysis of economic products and their relationships within the global production network. Maps industrial complexity, export capabilities, and productive knowledge.

  • Technique → Product Matrix
  • Capability mapping
  • Industrial complexity metrics

Firm Space

Mapping organizational structures and corporate entities within economic systems. Includes modeling templates for firm types, financial structures, and industrial sections.

  • Actor type templates
  • Corporate structure modeling
  • Financial sub-spaces (IB, UB, FoF, IF)

Literary Space

Exploration of literary works as social artifacts and their role in shaping cultural reality. Treats texts as epistemic objects encoding social knowledge.

  • Text-as-artifact analysis
  • Cultural representation mapping
  • Narrative structure indexing

05 — Data Model

Node Schema

The Sociognosis Index is built on a JSON Schema (data/idx/schema/schema.json) that defines the structure of each node in the epistemic graph.

Required Fields: Every node must specify id, name, category, description, functionalRoles, historicalContext, specific, relationships, metadata, and references.
// Core node structure (simplified) { "id": "node-bayesian-network", "name": "Bayesian Network", "category": "Epistemic Representation Form (Epistemic Blueprint)", "description": "A DAG representing probabilistic dependencies...", "functionalRoles": ["modeling", "inference"], "historicalContext": { "summary": "Developed in the 1980s...", "chronology": [ { "year": 1985, "event": "...", "context": "..." } ] }, "specific": { /* category-specific fields */ }, "relationships": [ { "type": "realizes", "target": "..." } ], "metadata": { "tags": ["probability", "graph"], "confidenceScore": 0.92, "createdAt": "2024-01-15T00:00:00Z" }, "references": [{ "title": "...", "link": "https://..." }] }

Relationship Types

Nodes are connected through typed relationships organized into families: ontological, causal, dependency, representational, epistemic, inferential, temporal, and others. The schema defines over 100 relationship types including:

instance_of part_of causes depends_on represents models derives_from enables predicts explains generates constrains

See schema.json → relationshipType enum for the complete list.

08 — Security

Known Risks & Limitations

The Sociognosis web interface is a static, client-side application under active development. The following security risks have been identified in the current codebase and are documented here for transparency. Until mitigations are in place, the interactive tools should only be used on trusted machines with trusted data.

Scope: These risks apply to the static site served from docs/—principally the Space Explorer (idx/index.html), the Node Editor (idx/edit.html), and the placeholder page (pending.html). No server-side backend is present; all processing occurs in the browser.
# Risk Severity Affected File Vector
1 Open Redirect Medium pending.html The ?back= query parameter is assigned to returnLink.href without validation. An attacker can craft pending.html?back=https://evil.com to redirect users to a malicious site—usable for phishing.
2 Cross-Site Scripting (XSS) High idx/index.html Node name, category, and shortName from data.json are injected into innerHTML without escaping (search results, category legend). Additionally, err.message is rendered unescaped on load failure. Any HTML in the data file executes in the user's browser.
3 Cross-Site Scripting (XSS) High idx/edit.html In suggested-relation rendering, targetNode.name, targetNode.id, and rel.targetNodeId are output without escHTML(). The data-id attribute on search results is likewise unescaped. An escHTML() helper exists but is not applied consistently.
4 API Token Exposure High idx/edit.html The DeepSeek API token can be persisted to localStorage. Combined with the XSS vulnerabilities above, any injected script can read localStorage.getItem('sociognosis-api-token') and exfiltrate it. The UI warns about localStorage accessibility, but the XSS holes make this actively exploitable.
5 Prompt Injection → XSS Chain High idx/edit.html AI responses are parsed via JSON.parse and merged into the node graph (Object.assign(selectedNode._data, improvedNodeData)). A crafted node description can induce the model to emit malicious field values, which then flow into the unescaped render paths (risks #2–#3), achieving script execution without directly editing data.json.
6 Missing Subresource Integrity (SRI) Medium All pages cdn.tailwindcss.com and Google Fonts are loaded without integrity= attributes. A compromise of either CDN would execute arbitrary JavaScript on every page that includes the script.
7 Tailwind Play CDN in Production Low All pages The Play CDN (cdn.tailwindcss.com) is explicitly marked by Tailwind as development-only. It ships a full runtime compiler, increases bundle size, and is not optimized or cached for production use.
8 No Content Security Policy (CSP) Medium All pages No Content-Security-Policy meta header is present on any page. A CSP would provide defense-in-depth against the XSS vulnerabilities by restricting inline scripts and unauthorized origins.
9 Reverse Tabnabbing Low pending.html A target="_blank" base directive opens links in new tabs without rel="noopener". A malicious page opened this way can access and manipulate the original tab via window.opener.

Attack Surface Summary

Highest Priority

The XSS vulnerabilities (risks #2–#3) are the most critical. They enable arbitrary script execution in the user's browser, which in turn makes the API token theft (#4) and prompt-injection chain (#5) fully exploitable. Fixing the rendering paths to consistently use escHTML()—or migrating to textContent / DOM APIs—would break the chain.

Defense-in-Depth

Adding a Content Security Policy (#8) and SRI hashes (#6) would limit the blast radius of any remaining injection flaws. Replacing the Tailwind Play CDN (#7) with a pre-built CSS file removes a runtime attack surface and improves performance.

Status: All risks listed above are acknowledged but unmitigated as of the current release. The interactive tools should not be exposed to untrusted users or fed untrusted data until the XSS paths are fixed and a CSP is in place.
